I built a full AI infrastructure and live trading system on a budget Android phone — and it actually works

By Bibhash — Independent Developer & Algorithmic Trader
June 2026  ·  Bhubaneswar, India
No laptop, no GPU, no desktop. Just a mid-range phone, a terminal emulator, and a lot of persistence.

Every guide about running local AI models assumes you have at least a dedicated GPU. Every trading automation tutorial assumes you have a Python environment on a proper computer. Every cloud deployment walkthrough assumes you have SSH from a real machine.

None of that applied to my situation. My entire development environment lives on a Motorola Edge 20 — a mid-range Android phone — running inside Termux. No laptop, no external keyboard dock, no GPU cluster. Just a phone, a terminal, and 8 GB of RAM doing work that most people would say requires far more.

The hardware constraint that shaped everything

Hard ConstraintNo pandas, numpy, yfinance, scipy, ta-lib

The aarch64 architecture forced every library choice. Packages that work trivially on x86 simply don't compile. Storage limits ruled out heavy dependencies entirely. This forced a discipline that turned out to be a feature — every component of the system is lean, purpose-built, and dependency-minimal.

Key insight: Constraints that seem like blockers often become the best design decisions. When you can't use pandas, you write pure Python that runs anywhere. When you can't install heavy deps, your system stays fast and portable.

Bibhash AI: a multi-model middleware running locally

Local AI Infrastructure

The centerpiece of the setup is a Flask-based middleware running on localhost:5000. Rather than picking a single AI provider, the system routes queries intelligently across multiple backends — cloud APIs for heavy reasoning tasks, and a locally-running quantized model for offline and low-latency use.

Fig 1 — Multi-model routing architecture running entirely on an Android phone
Multi-model routing layer
A single Flask server on port 5000 routes requests to Gemini, Claude, and DeepSeek cloud APIs — plus a locally running Qwen3 4B model via llama.cpp. The local model handles queries entirely on-device with no internet dependency.
Qwen3 4B running on-device via llama.cpp
A 4-billion-parameter quantized model runs in Termux on the phone's CPU. No GPU, no cloud call, no cost per query. Responses are slower than cloud APIs but fully private and available offline — on 8 GB RAM.
Two independent AI assistants, one device
A separate assistant called Aladdin runs on DeepSeek alongside Bibhash AI — two AI agents coexisting on the same phone for different workflow roles.

StockMind AI: a live trading scanner on GitHub Actions

Algorithmic Trading Automation

The trading side of the system is where constraint-driven engineering gets most interesting. Running a continuous stock scanner with no always-on machine required rethinking what "infrastructure" means entirely. The solution: GitHub Actions as a cron-driven compute layer, with Git itself acting as the persistent state database.

PS-OODA v3 signal engine
A 7-component weighted scoring model derived from a TradingView Pine Script indicator, fully ported to pure Python. Only signals scoring 8.0 or above are acted on — the 7.5–8.0 band was cut after historical analysis showed a 0% win rate in that range.
Smart Money Concepts (SMC) engine
A separate strategy module handles BTC, ETH, and Gold using SMC principles — swing points, market structure (HH/HL/LH/LL), Break of Structure, Order Blocks, Fair Value Gaps, and liquidity sweeps. All in pure Python, zero TA libraries.
Telegram-native command interface
The scanner doubles as a Telegram bot. Commands like /stats, /open, /best, /worst, /pause, and /resume let you query and control the live scanner in real time — from the same phone running the whole system.
Self-adjusting learning + GitOps config
A weekly adaptive tuner analyses trade outcomes and adjusts thresholds. Consecutive loss limits trigger auto-pause. Strategy parameters live in a committed config.json — updating a setting is a single git commit, no redeployment needed.

The trading methodology underneath it all

Framework

The signal engines aren't arbitrary. They're grounded in years of live XAUUSD and NSE equity trading using Smart Money Concepts — tracking institutional order flow, BOS/CHoCH patterns, and multi-timeframe confluence. The automation is built on top of a tested manual framework, not a backtested fantasy.

An adversarial audit of the historical trading journal identified premature fear-based exits as the primary profit leak. That finding is now baked directly into the signal logic — exits are rule-driven, not emotion-driven.

Study tools on the same stack

CA Foundation

The same phone also hosts study tools for CA Foundation exam prep — a 110-question interactive quiz bank covering Indian Contract Act, Sale of Goods, Partnership, LLP, and Companies Act, built as a self-contained HTML file with no framework. Same constraint philosophy: zero dependencies, works offline, loads instantly on mobile.

The full stack spans local AI inference, cloud API routing, live market scanning on free compute, Telegram-native control, and exam prep tooling — all maintained from a single Android device. No laptop required.

This is what happens when hardware limitations push you toward better engineering. Every constraint was a forcing function. Every workaround became a feature. The best code isn't the code written with the most resources — it's the code that survives without them.
